Por que essas decisões de código?

1. Interface User

Define a estrutura exata de um usuário. TypeScript garante que todos os objetos sigam essa estrutura.

2. Parâmetro userList: User[]

Especifica que a função recebe um array de usuários. Se você passar algo diferente, TypeScript avisa.

3. Retorno string[]

Garante que a função sempre retorna um array de strings. Você sabe exatamente o que esperar.

4. Tipagem explícita

filter((user: User) => ...) garante que 'user' seja do tipo User, permitindo acesso seguro às propriedades.
